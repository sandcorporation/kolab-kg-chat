# 적재 시 LLM 설명으로 임베딩을 강화하고, 질의이해 LLM을 제거한다 (Route C)

RAG 읽기 경로(ADR-0014)는 상품 임베딩 텍스트가 **상품명 + 속성 값**뿐이라 얇았다. 카탈로그 상품명이 대부분 영어라 한국어 질의가 임베딩 공간에서 상품과 안 붙었고(KO↔EN 미스매치, "플라스크"가 "Flask"를 못 찾음), 이를 메우려 **매 채팅 QueryAnalyzer(질의이해 LLM 1콜)** 로 번역·키워드를 뽑았다. 즉 얇은 인덱스를 질의당 반복 비용으로 땜질했다.

## 결정

embed-chat(GraphRAG)의 원리를 이식한다 — **지능을 질의 시점이 아니라 적재 시점으로 앞당긴다.**

- **적재 시 상품 임베딩을 LLM 설명으로 강화**한다(Route C): 상품마다 gpt-4o-mini가 **한/영 설명·키워드**를 생성해 임베딩 텍스트(상품명 + 속성 값 + 설명)에 넣는다. 설명은 content-hash로 게이팅·캐시(안 바뀐 상품 재호출 없음), 실패는 현행 텍스트로 폴백.
- **QueryAnalyzer(질의이해 LLM)를 제거**한다. HybridRetriever가 **raw 질의**로 키워드 ∪ 시맨틱 검색만 한다 — 강화된 인덱스가 KO/EN을 인덱스 측에서 커버하므로 질의당 번역이 불필요.
- 결과: **채팅당 LLM 2 → 1**(생성만). LLM 비용이 질의당 → 적재 1회로 이동해 분할상환된다.
- 강화는 항상 켜짐(플래그·thin 경로 없음, 전면 채택). ADR-0012(임베딩)·0014(RAG)를 **확장**한다.

## 이유 (실증)

- **recall A/B**(400 상품, 한국어 질의 15개, raw 질의·분석기 없음): 현행(thin) **2/15 → 강화 10/15**(5배). 분석기 포함 시 12→13. 강화가 어느 조합에서든 검색을 개선.
- **라이브**: "플라스크 추천해줘"가 현행 0 → 강화 **7 추천**. 서술형("동결 보존용 작은 병")·피펫·교반기 검색. diagnose 앵커링 없음.
- **비용**: 설명 생성 실측(상품당 입력~105·출력~82 tok) → 628k 1회성 표준 **~$42** / Batch API **~$21**. content-hash로 이후엔 변경분만.

## 대안 (기각)

- **강한 다국어 임베딩 모델(BGE-M3 등)로 교체**(Route B): 재임베딩 필요·미검증. 설명 강화가 데이터로 확인돼 우선.
- **분석기 유지(강화만)**: recall 13으로 최상이나 채팅당 2 LLM 유지. 비용·단순성 위해 제거 택함(recall 12→10 트레이드 수용).
- **프롬프트 캐싱으로 비용 절감**: 실측 결과 무의미 — 프롬프트가 ~78토큰이라 OpenAI 자동 캐싱 임계(1024토큰) 미달, 반복 호출에도 cached_tokens=0. 임계까지 패딩은 오히려 손해. **Batch API(50%)·content-hash 게이팅이 실제 레버**다.

## 결과

- 강화 backfill 명령(`embed_products`, 동시성 + content-hash 캐시). 대규모(수십만)는 **OpenAI Batch API**(비동기·24h·50%)가 후속.
- `.env`: `RAG_DESCRIPTION_MODEL`(기본 OPENAI_MODEL). 기존 `RAG_TOP_K`·`AGENT_TOKEN_BUDGET`·`AGENT_HISTORY_TURNS` 유지.
- 신규 테이블 `kg_description`(source_id → content_hash, description) — 설명↔임베딩 분리(모델 교체 시 재임베딩만).
- 원칙 유지: 임베딩은 **검색 리콜용**, 추천 카드의 그라운딩·가격·URL은 계속 **결정적** 부착(ADR-0001) — LLM이 상품 사실을 지어내지 않는다.
- 판정관 채점 A/B(강화 vs 현행 ± 분석기)는 recall→최종 품질 확증용 후속(권장).
- **(확장) PDF 문서 강화(옵션, `INGEST_PDF`)**: 상품 행의 직접-PDF URL을 적재 시 읽어(pypdf) 그
  스펙 텍스트를 설명 생성 프롬프트에 넣는다 — 지능을 적재 시점으로 더 앞당김. content_hash에
  pdf_url 포함(비면 무영향)·`is_current` fetch 게이트로 재처리 최소화. PDF 교체는 URL 변경으로
  감지. 상세: `.scratch/pdf-doc-enrichment/PRD.md`.
