# 반복 검색 루프로 질의생성(QueryAnalyzer)을 복구하고 만족까지 재시도한다

ADR-0015는 채팅당 LLM 콜을 줄이려 **질의생성(QueryAnalyzer)을 제거**하고 raw 질의로 한 번만
검색·선택했다(채팅당 LLM 1콜). 하지만 강화 임베딩에도 어려운 질의는 첫 검색어로 못 잡을 수
있고, 그때 `선택: 없음`으로 **그냥 끝나** 다른 각도로 다시 시도하지 못했다. 또 "그 중 첫 번째
어디 써?" 같은 팔로업에도 매번 새로 검색해 엉뚱한 결과를 냈다.

## 결정

채팅 에이전트를 **에이전틱 반복 검색**으로 바꾼다(ADR-0015를 품질 위해 되돌림).

- **QueryAnalyzer 복구·확장**: 원 질의 → 한/영 키워드 + 시맨틱 질의(KO/EN 미스매치 보완). 여기에
  **팔로업 라우팅**(이전 상품 참조면 검색 스킵)과 **재정식화**(거부 후보를 학습해 다른 검색어)를 더함.
- **만족까지 최대 N회 반복**(`AGENT_MAX_ITERS`, 기본 3): 검색 → 선택 → 불만족이면 재검색어 생성 후
  재시도. **만족 판정은 선택 단계를 재사용**한다(`선택: 없음`=불만족) — 별도 판정 LLM 없음.
- **스트리밍 엿보기**: 선택 응답을 스트리밍하며 첫 줄 `선택:`만 버퍼링해 판정. 만족(또는 마지막)이면
  나머지 근거를 토큰으로 흘리고, 불만족&잔여면 스트림을 끊어 출력 토큰을 아낀다.
- **종료**: 만족 · N 소진 · 무진전(재검색어 동일) · 재정식화 실패. 소진 시 정직한 폴백(억지 추천 없음).
- **새 기본값**(플래그 없음). prod는 재배포하지 않아 보호된다. `AGENT_MAX_ITERS=1`로 사실상 단일
  패스로 완화 가능.

## 이유

- **첫 시도 실패를 살린다**: 어려운 질의가 한 검색어로 안 잡혀도 다른 각도로 재시도 → 리콜↑.
- **팔로업 정합성**: 이전 상품을 가리키는 질문에 재검색하지 않고 대화 맥락으로 답한다.
- **판정 재사용**: 선택 LLM이 이미 "맞는 게 있나"를 판단하니, 그 `선택: 없음`을 불만족 신호로 쓰면
  판정 LLM을 따로 붙이지 않아도 된다.

## 대안 (기각)

- **별도 판정 LLM(Self-RAG)**: "후보가 충분한가"를 별도 콜로 판정 — 콜·복잡도↑. 선택 재사용으로 대체.
- **검색 점수 임계**: top-1 유사도로 만족 판정 — 임베딩 절대 점수는 신뢰도 낮아 부정확.
- **누적(union) 후보**: 이터레이션 결과를 모음 — 판정 일관성↓·복잡. 교체(fresh)로.
- **ADR-0015 유지(단일 패스)**: 비용 최소나 첫 시도 실패를 못 살림. 품질 위해 되돌림.

## 결과 / 트레이드오프

- **비용 증가(의도)**: 채팅당 LLM **1 → 2~6**. 팔로업 2 · 새검색 만족 2 · 재시도 1회 4 · N=3 소진 6.
  `AGENT_MAX_ITERS`로 상한 조절.
- 카드의 이름·가격·속성은 계속 소스 하이드레이션에서 결정적으로(ADR-0001/0016) — 루프는 검색
  리콜·정밀도만 높인다.
- 팔로업은 텍스트 답변만(카드 재표시는 프론트가 이전 상품 ID를 보내야 해 범위 밖).
- 품질 비교(단일 패스 vs 반복): `.scratch/iterative-retrieval/`에 기록.
