# 모든 Functional Attribute는 출처(provenance)와 신뢰도를 가진다

속성 추출은 LLM이 **자율로** 수행한다(사람 게이트 없음). 속성의 출처는 소스 컬럼, 상품 설명 텍스트, 그리고 OCR로 읽은 이미지로 신뢰도가 제각각이다. ADR-0001은 추천이 *어떤 요구조건을 충족하는지* 인용할 수 있어야 한다고 요구한다.

모든 Functional Attribute에 **출처(`source`: `structured` | `llm_text` | `llm_ocr`)** 와 **신뢰도(`confidence`)** 를 부여한다. LLM은 자율로 분류·추출해 곧장 그래프에 기록하며, provenance는 **게이트가 아니라 꼬리표**다.

이유:
- ADR-0001의 근거 품질을 끌어올린다 — "재질=PTFE 라서"가 *소스 확정값*인지 *LLM 추측*인지 구별해 인용할 수 있다.
- 자율 추출의 오류를 사후 감사·교정·가중할 수 있다(예: `llm_ocr` + 저신뢰 속성만 검수). 수집 속도는 늦추지 않는다.
- 보수적 사람-게이트(전부 cosmetic으로 두고 승격)는 기각했다 — 모든 것을 늦추고, 사용자가 자율 판정을 원했다.

## 결과

- 모든 속성 레코드가 `source`·`confidence`를 갖는다(처음부터 없으면 나중에 전체 재추출 필요 — 그래서 되돌리기 어려움).
- 검색/추천은 신뢰도로 가중·필터하고 출처를 인용할 수 있다.
- OCR 유래 속성은 본질적으로 저신뢰이며 `llm_ocr`로 구별된다(플랜 8번과 직결).
