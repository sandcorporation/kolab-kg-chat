# Apache AGE 그래프를 제거하고 상품 사실은 채팅 시 소스에서 하이드레이션한다 (C)

RAG(ADR-0014) + 강화 임베딩(ADR-0015)으로 오면서, Apache AGE 그래프(ADR-0003)는 사실상
**그래프 순회 없이** 노드/속성/가격 저장소로만 쓰였다(`find_compatible`·`COMPATIBLE_WITH`는
적재도 조회도 안 되는 죽은 코드였다). 즉 우리 DB는 소스 카탈로그를 **복제**하고 있었고,
그 복제를 최신으로 유지하려 동기화 서브시스템이 필요했다.

핵심 관찰: **상품 사실(이름·브랜드·가격·필드)은 이미 소스 DB에 있다.** 우리 DB가 반드시
가져야 할 것은 우리가 만든 것 — **임베딩과 LLM 설명**뿐이다. 나머지는 소스의 사본이다.

## 결정

**그래프(AGE)를 제거하고 "소스 하이드레이션"(C)으로 간다.**

- **우리 DB는 검색 색인만** 보관한다 — `kg_embedding`(source_id·name·content_hash·pgvector) +
  `kg_description`(source_id·content_hash·description). 상품/속성/변형 테이블 없음.
  `kg_embedding`이 곧 "적재된 상품" 인덱스이자 content-hash 게이트가 된다.
- **검색은 전부 우리 DB·인덱스에서**: 시맨틱(pgvector) ∪ 키워드(`name` **pg_trgm** ILIKE).
  질의당 소스 히트 0. 후보엔 이름 + 설명(kg_description)을 붙여 LLM이 고르게 한다.
- **상품 사실은 채팅 후반에만 소스에서 하이드레이션**한다. LLM이 고른 소수 id만
  `assemble_many(WHERE it_id IN …)`로 배치 조회 — 세 소스 테이블(item·option·field_info)이
  모두 `it_id` 인덱스를 타므로 K개 개별 조회보다 소스 부하가 낮다. 카드의 이름·가격·이미지·
  grounding 속성은 이 하이드레이션 문서에서 결정적으로 뽑는다(ADR-0001 유지 — LLM이 사실을
  지어내지 않는다).
- **적재는 임베딩·설명만** 만든다: 상품 → assemble → (설명 강화) 임베딩 저장. 속성은 임베딩
  텍스트 구성에만 쓰고 저장하지 않는다.
- **db 이미지에서 AGE 제거** — `pgvector/pgvector:pg16`(+ contrib `pg_trgm`). openCypher 불필요.

## 이유

- **무중복·항상 최신**: 카탈로그를 복제하지 않으므로 상품 데이터 동기화(스냅샷 드리프트)가
  사라진다. 하이드레이션은 항상 소스의 현재 값을 낸다. 우리 DB는 임베딩·설명만 갱신하면 된다.
- **단순성**: 그래프 저장소·openCypher·죽은 순회 코드·AGE 이미지 빌드가 통째로 사라진다.
- **인덱스 효율**: 소스는 손대지 않고(고객사 DB 변경 불필요) `it_id` IN 배치로 인덱스 스캔.
  검색은 로컬 pgvector·pg_trgm.

## 대안 (기각)

- **A. 관계형으로 물질화**(product/attribute/variant 3테이블): AGE만 걷어내고 사본은 유지.
  디커플링(읽기 시 소스 무관)은 지키지만 **중복·동기화가 남는다**. 소스를 채팅 때 쳐도 되는
  환경이면 C가 더 단순.
- **B. 강화 임베딩 행에 상품 페이로드까지 비정규화**: 단일 테이블로 스토어를 최소화하나,
  여전히 소스 사본을 우리 DB에 두는 셈. C는 아예 두지 않는다.

## 결과 / 트레이드오프

- **채팅이 소스 DB에 의존한다**: 하이드레이션이 매 채팅 소스를 친다(고른 소수만, it_id 인덱스).
  소스가 느리거나 죽으면 카드 구성이 영향받는다. 대신 후보 산출까지는 우리 DB만으로 되므로
  결합면은 "선택된 상품 하이드레이션"으로 국한된다.
- **적재 소스 = 채팅 소스**여야 한다: 임베딩을 만든 소스와 하이드레이션 대상 소스가 같은 DB를
  가리켜야 한다(`SOURCE_DB_*` 동일). 서로 다르면 카드가 비어 나온다.
- **배치 커밋 세션 제거**: 그래프 배치 세션이 사라져 임베딩 저장이 상품당 커넥션을 연다.
  임베딩 API 콜이 지배적이라 영향은 미미(후속 최적화 여지).
- ADR-0003(AGE 도입)·0008(상품 합류 동기화)의 상품-저장 전제를 대체한다. 임베딩 재도입
  (ADR-0012)·RAG(0014)·강화(0015)는 유지·확장.
