# 임베딩 대상과 모델

> **폐기됨(Superseded by ADR-0010).** 임베딩·시맨틱 검색은 시스템에서 제거되었다. 이 문서는 이력 보존용이다.

pgvector(ADR-0003)는 두 곳에서 쓰인다: (a) "비슷한 응용/환경"을 잇는 의미 다리(질문 9-A의 GraphRAG 핵심), (b) 통제 어휘 밖 롱테일 회수(하이브리드 어휘). 무엇을 벡터화하고 어떤 모델을 쓸지 정한다.

## 결정

- **임베딩 대상**: ① **Application·Condition 노드**(이름+정의) — "초저온 보관"↔"극저온 저장"을 잇는 다리. ② **조립된 Product의 서술 텍스트**(설명 + OCR 텍스트) — 통제 어휘 밖 롱테일 회수.
- **모델**: OpenAI **`text-embedding-3-small`** 로 시작(한국어 무난·저렴), 회수율 부족 시 `-large`로 승격.
- 모든 임베딩에 **모델/버전 + 스냅샷**을 태깅한다.

## 이유

Application·Condition 노드를 임베딩해야 질문 9-A의 의미 다리가 성립한다 — Product 텍스트만 임베딩하면 "비슷한 응용"을 못 잇는다. `-small`은 한국어에 무난하고 저렴하며, 버전 태깅이 모델 교체의 되돌리기 비용을 낮춘다.

## 결과

- 모델/차원 변경은 전체 재임베딩이 필요 → 모델 버전 태그로 **점진 마이그레이션**(바뀐 것만 재임베딩).
- 임베딩도 provenance(ADR-0004)·content-hash 스냅샷(ADR-0008)과 함께 추적된다.
