# 이미지 속성은 비전 LLM 직독으로 추출한다 (고전 OCR 대신)

kolabshop 상품에서 결정적 Functional Attribute의 **약 절반은 이미지 안에만** 존재한다 — 스펙표, 치수 도면, 데이터시트가 이미지로 박혀 있고 한국어 텍스트가 많다. 따라서 이미지 추출은 보조가 아니라 1급 경로다.

이미지에서 속성을 추출할 때 **비전 가능한 LLM이 이미지를 직접 읽어**(이미지 → 속성) 추출한다. 고전 OCR→텍스트→LLM 2단계는 쓰지 않는다. 처리 전 **트리아지**(싼 1차 판정)로 스펙/표/도면 이미지만 거르고 마케팅 사진은 건너뛴다. 이미지 유래 속성은 `source=llm_ocr`로 태깅한다(ADR-0004).

이유: 고전 OCR은 표·도면의 구조(셀 정렬·헤더)를 무너뜨리고 오류 단계를 하나 더 더한다. 비전 LLM은 레이아웃을 이해하고 이미지 속 한국어에 강하며, 유형별 통제 어휘로의 속성 추출(ADR-0001)로 한 번에 간다. 트리아지는 비용·노이즈를 통제한다.

## 결과

- 이미지당 비전 LLM 비용 발생 — 트리아지와 "텍스트에서 이미 확정된 속성은 OCR 생략"으로 호출 수를 억제한다.
- `llm_ocr` provenance 덕에 추천이 이미지 유래 값을 가중·표시할 수 있다.
- 결정적 데이터의 ~50%가 이미지뿐이므로, 비전 LLM 가용성·비용은 이 제품의 핵심 운영 의존성이다.
